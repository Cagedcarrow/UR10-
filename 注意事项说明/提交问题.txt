既然你已经成功打通了 **SSH 通道**并创建了 **MATLAB 工程 (Project)**，以后的代码管理将变得非常简单。你不需要再重复配置 SSH 密钥或修改远程地址，只需要在日常开发中遵循“**保存 - 提交 - 推送**”的流程即可。

以下是今后管理代码的标准化操作：

### 1. 日常提交与同步流程

每当你完成了一段代码修改（例如改进了 UR10 的轨迹算法），请按照以下顺序操作：

* **提交更改 (Commit)**：
* 在 MATLAB 顶部的 **工程 (Project)** 选项卡中，点击 **源代码管理 (Source Control)** 组里的 **提交 (Commit)**。
* 在弹出的窗口中勾选你修改的文件，并在注释框中写下具体的修改内容（例如：“优化了逆运动学收敛速度”）。
* 点击 **提交**。这会将你的修改记录在本地电脑的 Git 仓库中。


* **推送到云端 (Push)**：
* 在 MATLAB 下方的 **Git 面板**中，点击带有绿色向上箭头的 **推送 (Push)** 按钮。
* 由于你已经配置了 SSH 密钥，MATLAB 会自动完成身份验证并上传，不再需要输入密码。



### 2. 保持代码最新 (Pull)

如果你在 GitHub 网页端手动修改了代码（比如编辑了 README 文件），或者在另一台电脑上上传了新版本，你需要同步到当前电脑：

* 在 Git 面板点击 **拉取 (Pull)**。这会将 GitHub 上的最新版本下载并合并到你当前的本地文件夹中。

---

### 3. 多分支管理建议

由于你目前存在 `main` 和 `master` 两个分支，为了避免混淆，建议你固定使用一个分支：

* **查看当前分支**：在 MATLAB 底部 Git 面板的左侧，会显示 **当前分支: master**。
* **推荐做法**：建议你在 GitHub 网页端将 `master` 设置为**默认分支 (Default Branch)**，这样以后打开仓库一眼就能看到最新的代码。

---

### 4. 关键文件管理

* **工程文件 (.prj)**：这是 MATLAB 管理 Git 状态的核心，请务必保证它始终在仓库中。
* **忽略文件 (.gitignore)**：如果你产生了新的临时文件（如 `.asv`），可以随时右键该文件 -> **源代码管理** -> **忽略**，防止杂乱文件上传到 GitHub。

### 5. 如果遇到网络波动

如果再次遇到“操作超时”：

1. 检查你的手机热点是否开启（移动网络通常比校园网/宽带更稳定）。
2. 确保你没有关闭或删除 `.ssh` 文件夹中的 `id_rsa` 文件。
